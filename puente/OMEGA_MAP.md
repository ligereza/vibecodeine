# PUENTE Ω↔FLUJO — mapa conceptual

idea_generativa (en Desktop) es la teoría del motor: el sistema Ω que describe cómo arranca, se sobre-narra y se cierra un proceso creativo. flujo es el motor corriendo con clientes reales, con pedidos, plazos y entregas de verdad. Este mapa es el puente entre los dos, fechado 12-jul-2026.

| Ω | ↔ | flujo |
|---|---|---|
| ⊕ semilla (residuo fechado que arranca todo) | ↔ | inbox/ pedidos + datadrops/ fechados |
| ⚙️ Motor 5 pasos (semilla→precipitación→sobre-narración→Ω11→registro) | ↔ | pipeline intake→job→brief→render→entrega |
| 🧪 Precursor (baja umbral, no añade contenido) | ↔ | templates JSON knowledge/ + Makefile |
| 🌀 Psicosis (sobre-narración hasta bifurcación) | ↔ | multi-agente (Claude director / Gemini intérprete / Qwen bruto) |
| 👁️ Apofenia (patrón espurio, sobreajuste) | ↔ | sobre-ingeniería meta (airdrop, docs) — autodiagnosticada en critique_acida |
| 🎯 Ω11 falsabilidad (pierde-si declarado ex ante) | ↔ | flujo verify + pytest + hub_smoke |
| ⚓ Ancla/techo (descanso como operación) | ↔ | checkpoints/ + .archive/ + noisette congelado |
| 🌍 Cultura (regularizador externo, prior p) | ↔ | clientes reales + linea_editorial v4.1 + portfolio público |
| 📜 Estatuto declarado (teorema/defendible/analogía/metáfora/insostenible) | ↔ | schemas Pydantic / manifest |
| 〰️ Tilde (residuo intraducible, NO ahorro de tokens) | ↔ | brechas ES↔EN↔código: Gemini→Claude proxy, spanish-optimizer, brief≠pedido |

## Cómo usar

- Al diseñar una feature nueva, buscá su equivalente Ω.
- Si no tiene, sospechá inercia.
- Si un concepto Ω no tiene equivalente en flujo, es un ⊕ pendiente.
