# HOTFIX — v0.34.12 Windows UTF-8 en CLI

Fecha: 2026-06-23

## Motivo

Después de aplicar v0.34.11 en Windows/Git Bash, `run_airdrop_checks.py` avanzó correctamente por:

- validación de `_airdrop/`
- dry-run
- apply
- `pip install -e .[dev]`
- `compileall`
- `pytest`

pero falló en `python -m flujo health` al imprimir el símbolo de advertencia `⚠` con consola Rich en modo Windows legacy/cp1252:

```text
UnicodeEncodeError: 'charmap' codec can't encode character '\u26a0'
```

## Cambio

`src/flujo/cli.py` ahora reconfigura `stdout` y `stderr` a UTF-8 con `errors="replace"` en Windows antes de construir el `Console` global de Rich.

Esto evita fallos al imprimir:

- `⚠`
- `✓`
- flechas `→`
- símbolos del changelog
- textos con Unicode usados por el CLI

## Validación esperada

Tras aplicar este airdrop, reejecutar:

```bash
py scripts/run_airdrop_checks.py "v0.34.12 - hotfix windows utf8 cli"
```

Debe completar `health`, `version`, checkpoint y push.

## Nota

El airdrop v0.34.11 ya había aplicado los cambios antes de fallar en health. Este hotfix debe aplicarse encima; el checkpoint final debería incluir ambos conjuntos de cambios.
