# HOTFIX — v0.34.13 Workflow hardening

Fecha: 2026-06-23

## Resumen

Se agregan guardrails para que el workflow de airdrops y verificación sea más difícil de romper.

## Cambios

- Nuevo comando `flujo verify`:
  - `compileall`
  - `pytest`
  - `flujo health`
  - `flujo version`
  - `scripts/hub_smoke.py`
- Nuevo `scripts/hub_smoke.py`:
  - levanta `flujo serve` en puerto libre
  - valida `/api/ping`
  - valida que `/api/events` no bloquee otras APIs
  - valida que traversal y archivos internos (`/pyproject.toml`, `/src/flujo/cli.py`) no sean servidos
  - valida que assets públicos permitidos como `/projects/flujo/flujo.json` sí respondan
- CI ahora corre en Ubuntu y Windows con `python -m flujo verify`.
- `scripts/run_airdrop_checks.py` agrega `--resume` para continuar checks/checkpoint si un airdrop ya fue aplicado y falló después.
- El hub ya no sirve todo el repo como static fallback: solo `context/`, `svg/`, `docs/`, `projects/flujo/`, `projects/plano/`, `projects/tapiz/` y `datadrops/` por su endpoint dedicado.

## Comandos nuevos

```bash
py -m flujo verify
py -m flujo verify --no-hub-smoke
py scripts/hub_smoke.py
py scripts/run_airdrop_checks.py --resume "mensaje"
```

Linux/macOS: usar `python3` en lugar de `py`.
