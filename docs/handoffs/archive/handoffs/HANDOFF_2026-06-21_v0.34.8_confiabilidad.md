# HANDOFF — v0.34.8 Confiabilidad de airdrop, base universal y setup

Fecha: 2026-06-21

## Resumen

Airdrop de mantenimiento para cerrar hallazgos de revisión técnica:

1. Base universal de `config.json` segura ante comillas/texto humano.
2. Rollback de airdrop completo: restaura `REPLACE` y elimina archivos `NEW`.
3. `flujo airdrop apply` valida `_airdrop/` antes de aplicar.
4. Alias real `flujo app` → `flujo serve`.
5. Fallback YAML simple reconoce listas escalares.
6. `setup.sh` / `Makefile` más portables entre Windows y Linux.
7. Versionado actualizado a `0.34.8`.

## Importante para aplicar este airdrop

Este airdrop modifica `src/flujo/airdrop.py` (el motor de airdrops). Por eso el
validador conservador actual requiere autorización explícita.

Como el flag nuevo de `run_airdrop_checks.py` todavía no existe antes de aplicar
este airdrop, usar esta secuencia para esta vez:

```bash
# 1) Descomprimir el ZIP en la raíz del repo. Debe quedar una carpeta _airdrop/

# 2) Validar permitiendo explícitamente el cambio al motor de airdrops:
py scripts/validate_airdrop.py --allow-airdrop-engine

# 3) Simular:
flujo airdrop dry-run

# 4) Aplicar con la CLI actual:
flujo airdrop apply "v0.34.8 - confiabilidad airdrop y base universal"
```

Después de aplicado, ya queda disponible para futuros airdrops:

```bash
py scripts/run_airdrop_checks.py "vX.Y.Z - descripcion" --allow-airdrop-engine
flujo airdrop apply "mensaje" --allow-airdrop-engine
```

## Cambios por archivo

- `src/flujo/render/piezas.py`
  - `_create_universal_base()` deja de construir JSON con f-string.
  - Ahora crea un dict y escribe con `json.dumps(..., ensure_ascii=False, indent=2)`.
  - Agrega `_strip_wrapping_quotes()` para no borrar comillas legítimas al parsear valores simples del brief.

- `src/flujo/airdrop.py`
  - Agrega manifest `_airdrop_manifest.json` en cada backup.
  - `rollback_last()` ahora:
    - restaura archivos `REPLACE`,
    - elimina archivos `NEW`,
    - limpia carpetas vacías creadas por archivos nuevos sin borrar raíces protegidas.
  - Mantiene compatibilidad con backups antiguos sin manifest.

- `src/flujo/cli.py`
  - `flujo airdrop apply` ejecuta `scripts/validate_airdrop.py` antes de aplicar.
  - Agrega `--skip-validation` y `--allow-airdrop-engine`.
  - Corrige alias: `flujo app` ahora existe realmente.

- `src/flujo/jobs/brief.py`
  - El parser YAML simple detecta listas escalares (`productos`, `pendientes`) cuando no está disponible PyYAML.

- `scripts/run_airdrop_checks.py`
  - Agrega `--allow-airdrop-engine` y lo pasa al validador.

- `scripts/setup.sh`
  - Detecta `PYTHON_BIN`, `py`, `python3` o `python`.
  - Mejora mensajes posteriores al setup.

- `Makefile`
  - `PYTHON ?= python3` para permitir override.
  - `make test` instala editable con dev deps antes de correr tests.

- `README.md`, `pyproject.toml`, `src/flujo/version.py`
  - Versión `0.34.8` y nota de validación para airdrops que tocan el motor.

- Tests nuevos/actualizados:
  - `tests/test_render_formats.py`: regresión para comillas en base universal.
  - `tests/test_airdrop.py`: rollback elimina `NEW` y restaura `REPLACE`.
  - `tests/test_cli_smoke.py`: `flujo app --help` existe.
  - `tests/test_jobs_brief.py`: parser YAML simple soporta listas escalares.
