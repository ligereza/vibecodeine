# HOTFIX — v0.34.10 Runner airdrop: import path Windows/Git Bash

Fecha: 2026-06-22

## Problema

`py scripts/run_airdrop_checks.py "mensaje"` podía fallar en Windows/Git Bash con:

```txt
No module named 'flujo.airdrop'; 'flujo' is not a package
```

Causa: al ejecutar un script dentro de `scripts/`, Python deja `scripts/` en
`sys.path`. El runner además priorizaba `scripts/` para importar
`validate_airdrop.py`. Entonces `import flujo.airdrop` podía resolver primero
`scripts/flujo.py` como módulo top-level `flujo`, en vez del paquete real
`src/flujo/`.

## Solución

- `scripts/run_airdrop_checks.py`
  - Agrega `_ensure_src_first()` para mantener `src/` por delante de `scripts/`.
  - Carga `scripts/validate_airdrop.py` por ruta usando `importlib.util`, sin
    meter `scripts/` al inicio de `sys.path`.
  - Refuerza `_airdrop_dry_run`, `_airdrop_apply` y `_checkpoint` llamando
    `_ensure_src_first()` antes de importar `flujo.airdrop`.

- `tests/test_run_airdrop_checks.py`
  - Agrega regresión para simular `scripts/` antes que `src/` en `sys.path`.
  - Verifica que `flujo.airdrop` sigue cargando desde `src/flujo`.

- Versionado/documentación
  - Bump a `0.34.10` en `src/flujo/version.py` y `pyproject.toml`.
  - README / guía / contexto / CLI actualizados a v0.34.10.

## Pruebas ejecutadas en clon limpio actual de GitHub

```bash
python -m compileall -q src scripts tests
python -m pytest tests/ -q --tb=short
python -m flujo health
python -m flujo version
git diff --check
```

Además se ejecutó una prueba real del runner con un airdrop temporal y:

```bash
python scripts/run_airdrop_checks.py "probe runner import path" --skip-checkpoint
```

Resultado: OK.

## Cómo aplicar este hotfix

Como este hotfix arregla precisamente el runner, **no uses el runner para aplicar
este ZIP**. Usa la ruta manual/CLI:

```bash
py scripts/validate_airdrop.py
flujo airdrop dry-run
flujo airdrop apply "v0.34.10 - hotfix runner import path"
```

Si `flujo` no está instalado como comando:

```bash
py -m flujo airdrop dry-run
py -m flujo airdrop apply "v0.34.10 - hotfix runner import path"
```

Después de aplicado, el runner debería volver a funcionar:

```bash
py scripts/run_airdrop_checks.py "probe futuro" --skip-checkpoint
```
