# HANDOFF - v0.35.9 Windows checkout + issue titles hotfix

ASCII-only file. Owner uses Windows + Git Bash. Use `py`, not `python`.

## Problem fixed

GitHub Actions Windows checkout failed before tests because three checkpoint filenames were too long.
The issue templates also still showed generic request titles instead of area-specific EVENTOS / SUPLEMENTOS.

## What changed

1. Long checkpoint files were renamed to short names in the repo commit.
2. Old generic issue templates were removed from the repo commit:
   - `.github/ISSUE_TEMPLATE/pedido_diseno.yml`
   - `.github/ISSUE_TEMPLATE/pedido_impresion.yml`
3. New area templates were added:
   - `.github/ISSUE_TEMPLATE/pedido_eventos.yml` with title `[EVENTOS] `
   - `.github/ISSUE_TEMPLATE/pedido_suplementos.yml` with title `[SUPLEMENTOS] `
4. Gmail bridge now creates issue titles:
   - `[EVENTOS] subject`
   - `[SUPLEMENTOS] subject`
5. Workflows use `actions/checkout@v5`.
6. Added cleanup helper for airdrop users:
   - `scripts/cleanup_v0359_windows_paths.py`

## IMPORTANT if applying by airdrop

Airdrop copy does not delete old files. After applying the airdrop, run:

```bash
py scripts/cleanup_v0359_windows_paths.py
```

Then run normal checks:

```bash
py scripts/validate_airdrop.py
py scripts/run_airdrop_checks.py "v0.35.9 - windows checkout issue titles"
```

If you are applying by normal git commit/merge, the deletes/renames are already in the commit.
