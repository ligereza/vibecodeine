## Why v0.35.7 exists

The old LAST_HANDOFF had mojibake / broken letters from encoding issues.
Although CLI UTF-8 handling was improved earlier, this file is read by humans and agents across Windows/Git Bash and web UIs.
To avoid future breakage, LAST_HANDOFF is now deliberately ASCII-only.

Rules for future agents:

- Keep `context/LAST_HANDOFF.md` ASCII-only.
- Use `py -m flujo ...` in owner-facing docs and handoffs.
- Code identifiers, logs, and CLI internals can be English/ASCII.
- Avoid special terminal symbols if not needed.
- Do not store tokens or Gmail credentials in repo.
