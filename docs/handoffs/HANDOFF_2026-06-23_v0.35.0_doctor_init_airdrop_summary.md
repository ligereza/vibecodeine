# HANDOFF — v0.35.0 Doctor, init fresh y resumen de airdrops

Fecha: 2026-06-23

## Resumen

Primer avance hacia 0.35: herramientas de soporte para que el repo sea más fácil de diagnosticar, preparar y cerrar después de un airdrop.

## Cambios

- Nuevo comando `flujo doctor`:
  - diagnostica versión, Python, encoding, repo root, workspace, carpetas, DB, `_airdrop/`, Git branch/origin/status y puertos locales.
  - No corre tests pesados; es para revisar una máquina nueva o un clon fresco.
- `flujo init --fresh`:
  - prepara `jobs/_template`, `inbox/`, `data/`, `datadrops/`, `datadrops/incoming/`.
  - reconstruye índice SQLite por defecto; se puede omitir con `--no-rebuild-index`.
- `scripts/run_airdrop_checks.py`:
  - agrega un resumen final con versión, rama, commit, mensaje, checks corridos, git status y próximo paso recomendado.
- Nuevos tests smoke para `doctor` e `init --fresh`.

## Comandos útiles

```bash
py -m flujo doctor
py -m flujo init --fresh
py -m flujo init --fresh --no-rebuild-index
py -m flujo verify
```

Linux/macOS: usar `python3` en lugar de `py`.
