# Handoff — v0.34.7 fix runner Windows/Git Bash

Fecha: 2026-06-21
Versión destino: 0.34.7

## Objetivo

Reparar la promesa de v0.34.5/v0.34.6: `py scripts/run_airdrop_checks.py "mensaje"` debe funcionar en Windows/Git Bash sin invocar `bash` desde Python.

## Problema

El runner anterior ejecutaba internamente:

```text
bash scripts/apply_airdrop.sh --dry-run
bash scripts/apply_airdrop.sh --apply
bash scripts/checkpoint.sh "mensaje"
```

En Windows/Git Bash, al llamarlo desde Python, el entorno resolvió mal `bash` y falló con `execvpe(/bin/bash) failed`.

## Cambio

- `scripts/run_airdrop_checks.py` ahora usa Python puro:
  - `flujo.airdrop.scan_airdrop()` para dry-run.
  - `flujo.airdrop.apply_airdrop()` para aplicar.
  - `flujo.airdrop.run_auto_checkpoint()` para checkpoint/push.
- Mantiene `pip install`, `compileall`, `pytest`, `health`, `version` y changelog check.
- Mantiene logs en `_logs/airdrop_error_*.txt` si algo falla.
- Agrega tests estáticos para impedir volver a invocar `bash` para apply/checkpoint.

## Pruebas realizadas

```bash
python -m pip install -e ".[dev]"
python -m compileall -q src scripts tests
python -m pytest tests/ -q
python -m flujo health
python -m flujo version
```

Resultado: OK en clon de reparación.

## Uso futuro

```bash
py scripts/validate_airdrop.py
py scripts/run_airdrop_checks.py "vX.Y.Z - descripcion"
```

Si falla, enviar `_logs/airdrop_error_*.txt`.
